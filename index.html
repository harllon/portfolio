Projects

Color App

Esse aplicativo foi desenvolvido em Kotlin e permite que o usuário visualize cores que podem ser escolhidas de três formas diferentes. A primeira consiste em escolher as componentes RGB da cor por meio de sliders, enquanto a segunda consiste em escrever os componentes RGB em hexadecimal. Por fim, a terceira consiste em escolher as componentes HSV por meio de "sliders". Os modos de visualização e de mudança nos valores das cores são sincronizados utilizando "Live Datas". Há um botão de ajuda no aplicativo que abre uma "dialog box" explicando o próprio aplicativo. É possível realizar login e cadastro, para os quais foi utilizado o Firebase, com o intuito de que o usuário possa salvar suas cores preferidas e visualizá-las depois. O usuário pode navegar entre as possíveis formas de visualizar as cores e a tela com as cores salvas utilizando tabs no inferior da tela, que foram confeccionadas utilizando "viewPager". As cores salvas são mostradas na Tab correspondente, utilizando-se de "Recycler View" aliado ao "Room Database" que armazena as cores salvas pelo usuário. Além disso, os itens da "Recycler View" – no caso, as cores – são selecionáveis, de forma que, ao selecionar uma cor, uma "dialog box" é aberta, mostrando informações sobre a cor salva.

Detalhes de código? Confira o github do projeto: Color App Github.

Sensor App

Esse aplicativo, desenvolvido em Java, permite a coleta de dados de diversos sensores disponíveis nos celulares, como, por exemplo, o acelerômetro e o giroscópio. O usuário pode selecionar vários sensores de uma lista de opções, assim como pode selecionar alguns métodos oriundos dos dados dos sensores como velocidade, altitude e verificação de movimento. Ao terminar as seleções e iniciar a coleta de dados, os sensores escolhidos são ativados. Esse gerenciamento, desde saber quais os sensores selecionados até ativá-los, foi feito utilizando Live Datas e a biblioteca de sensores disponível na documentação do Android. Ao fim da coleta, caso o usuário queira salvar os dados, eles serão escritos em arquivos de extensão “.csv”, sendo um arquivo para cada sensor e método selecionado. Para viabilizar esse processo, foi utilizada a Room Database de forma a conseguir armazenar todos os dados que o usuário deseja salvar. O usuário também pode compartilhar os arquivos usando outros softwares, incluindo e-mail, uma vez que, os arquivos salvos serão automaticamente anexados ao corpo da mensagem.

Detalhes de código? Confira o github do projeto: Sensor App Github.

Dataset de Light Fields Sintéticos

Esse projeto foi desenvolvido em meu período de iniciação científica, cujo objetivo era gerar um dataset de vídeos light field sintéticos que se tornassem desafios para os algoritmos de compressão de vídeo. Para produzir este projeto, foram criados cenários utilizando os softwares Blender e Unity, além de um plugin para o Blender, responsável por criar uma câmera plenóptica. Os cenários eram criados aplicando texturas nos objetos da cena e criando zonas de oclusão entre os objetos, duas técnicas que dificultam a compressão de vídeos. A primeira dificulta a compressão "intra-frames" – isto é, dentro de cada "frame" do vídeo –, pois essa compressão busca similaridades entre os pixels da imagem e a textura cria uma diferenciação significativa entre os pixels próximos da imagem. A segunda técnica dificulta a compressão "inter-frames", isto é, entre os frames, pois é difícil predizer com precisão onde um determinado pixel vai aparecer, especialmente se ele entrar em uma zona de oclusão. Ao fim da captura, os vídeos foram visualizados com um visualizador de vídeo light fields que permitia explorar as duas principais propriedades dessa forma de vídeo/imagem, que são a mudança de foco após a filmagem e a mudança de ponto de vista. Esse projeto teve um artigo publicado no SIBGRAPI (Conference on Graphics, Patterns and Images), em 2020, tendo sido realizada uma apresentação no mesmo congresso.

Quer saber mais? Dê uma olhada na página do projeto: Dataset Light Field Sintéticos.
